 ADR 001: Use PlantUML for UML Diagrams

## Status
Accepted

## Context
The project requires creating UML diagrams that are easy to version-control and review in GitHub.

## Decision
PlantUML is chosen for creating UML diagrams because it uses text-based source files that work well with Git version control.

## Consequences
- Diagrams can be easily reviewed using pull requests.
- Changes are diff-friendly.
- Team members need basic knowledge of UML syntax.
